# Environment Variables for Render WebSocket Service

Copy these into Render â†’ Environment Variables section

## Required Variables

**NODE_ENV**
```
production
```

**PORT**
```
10000
```

**HOSTNAME**
```
0.0.0.0
```

**CORS_ORIGIN**
```
https://boardly.online,https://www.boardly.online
```
Note: Add all domains that will connect to your WebSocket server, comma-separated

**DATABASE_URL**
```
postgresql://postgres.xxxxx:[YOUR-PASSWORD]@aws-0-eu-central-1.pooler.supabase.com:5432/postgres
```
Note: Replace with your actual Supabase connection string

---

## Service Configuration

**Build Command:**
```
npm install && npm run db:generate
```

**Start Command:**
```
npm run socket:start
```

**Health Check Path:**
```
/health
```

**Auto-Deploy:** Yes (enabled)

---

## Notes

- Database URL should be the same as in Vercel
- CORS_ORIGIN must include your production domain
- After changing variables, manually redeploy the service
- Health check endpoint returns `{"ok":true}` when healthy
