# Environment Variables Template for Vercel

Copy and paste these into Vercel → Settings → Environment Variables

## Required Variables

```
DATABASE_URL
```
Value: Your Supabase PostgreSQL connection string
Example: `postgresql://postgres.xxxxx:password@aws-0-eu-central-1.pooler.supabase.com:5432/postgres`

---

```
NEXTAUTH_SECRET
```
Value: Random 32+ character string (generate with: `openssl rand -base64 32`)

---

```
JWT_SECRET
```
Value: Random 32+ character string (generate with: `openssl rand -base64 32`)

---

```
NEXTAUTH_URL
```
Value: `https://boardly.online`

---

```
RESEND_API_KEY
```
Value: Your Resend API key (starts with `re_`)

---

```
EMAIL_FROM
```
Value: `Boardly <noreply@boardly.online>`

---

```
NEXT_PUBLIC_SOCKET_URL
```
Value: Your Render WebSocket URL
Example: `https://boardly-websocket.onrender.com`

---

## Optional Variables

```
NEXT_PUBLIC_SUPABASE_URL
```
Value: Your Supabase project URL
Example: `https://xxxxxxxxxxxxx.supabase.co`

---

```
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY
```
Value: Your Supabase anon/public key

---

## OAuth (Optional - configure later)

```
GOOGLE_CLIENT_ID
```
Value: From Google Cloud Console

---

```
GOOGLE_CLIENT_SECRET
```
Value: From Google Cloud Console

---

```
GITHUB_CLIENT_ID
```
Value: From GitHub Developer Settings

---

```
GITHUB_CLIENT_SECRET
```
Value: From GitHub Developer Settings

---

## Notes

- All variables should be set for **Production**, **Preview**, and **Development** environments
- After adding variables, redeploy your project
- Secrets should NEVER be committed to git
